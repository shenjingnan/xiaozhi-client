## 前置条件
- Dashboard 页面已打开且加载完成
- 已存在至少一个可用的 MCP 接入点（端点状态为"已连接"或"未连接"均可）

## 测试步骤
1. 打开页面 http://localhost:9999/dashboard
2. 等待页面加载完成
3. 找到并点击接入点设置按钮（齿轮图标按钮，位于导航栏或状态卡片区域）
4. 截图（接入点设置对话框打开状态）
5. 验证对话框标题为"配置小智服务端接入点"
6. 验证对话框描述包含"点击保存后，需要重启服务才会生效"
7. 验证至少显示一个接入点条目，每个条目包含：
   - 接入点地址文本（截断显示格式）
   - 连接状态 Badge（显示"已连接"/"未连接"/"操作中"）
   - 复制按钮
   - 连接/断开切换按钮
   - 删除按钮
8. 记录第一个接入点的当前连接状态（"已连接"或"未连接"）
9. 截图（操作前状态）

### 场景 A：如果当前状态为"未连接"，执行连接操作
10. 点击该接入点的连接按钮（Wifi 图标，title 为"连接"）
11. 验证按钮进入"操作中"状态（显示旋转的 Loader 图标，文字变为"操作中"，按钮禁用）
12. 等待操作完成
13. 截图（连接操作完成后）
14. 验证状态 Badge 变更为"已连接"（绿色背景 + Wifi 图标）
15. 验证出现 toast 提示包含"接入点连接成功"

### 场景 B：如果当前状态为"已连接"，执行断开操作
10. 点击该接入点的断开按钮（WifiOff 图标，title 为"断开连接"）
11. 验证按钮进入"操作中"状态（显示旋转的 Loader 图标，文字变为"操作中"，按钮禁用）
12. 等待操作完成
13. 截图（断开操作完成后）
14. 验证状态 Badge 变更为"未连接"（灰色背景 + WifiOff 图标）
15. 验证出现 toast 提示包含"接入点断开成功"

### 恢复步骤（确保测试环境一致性）
16. 如果执行了断开操作，再次点击连接按钮恢复原始连接状态
17. 如果执行了连接操作，点击断开按钮恢复原始断开状态
18. 等待恢复操作完成
19. 关闭接入点设置对话框
