## 前置条件

- 后端服务已启动（http://localhost:9999）
- 当前已登录 Web UI 界面
- 注意：本测试验证 SSE 日志流式推送的完整交互流程，**不会实际执行 NPM 安装**（避免修改系统环境），重点验证前端与后端 SSE 端点的通信正确性

## 测试步骤

### Part 1：版本升级对话框 → 触发安装

1. 打开页面 http://localhost:9999/dashboard
2. 等待页面加载完成
3. 截图（Dashboard 初始状态）
4. 验证导航栏右侧存在"升级版本"按钮（带有下载图标）
5. 点击"升级版本"按钮
6. 截图（版本选择对话框打开状态）
7. 验证对话框标题为"选择安装版本"
8. 验证对话框包含"正式版"版本类型选项（默认选中）
9. 等待版本列表加载完成（下拉框不再显示"正在获取版本列表..."）
10. 截图（版本列表加载完成状态）
11. 在版本选择下拉框中选择一个可用版本（非当前版本）
12. 验证版本已被选中（下拉框显示选中的版本号）
13. 点击"确定安装"按钮
14. 截图（点击安装后的状态）

### Part 2：SSE 安装日志对话框验证

15. 验证版本选择对话框自动关闭
16. 验证安装日志对话框（InstallLogDialog）自动弹出
17. 截图（InstallLogDialog 打开状态）
18. 验证对话框标题为"正在安装"
19. 验证存在状态徽章，显示"正在安装..."文字（带脉冲动画的终端图标）
20. 验证进度条存在且进度值大于 0（表示安装进行中）
21. 验证状态描述区域显示"正在安装..."文字
22. 验证存在"安装日志"可折叠区域
23. 点击"安装日志"展开按钮
24. 截图（安装日志展开状态）
25. 验证日志区域显示终端图标和"等待日志输出..."提示（初始状态）

### Part 3：SSE 连接与数据流验证

26. 打开浏览器开发者工具 → Network 标签页
27. 在 Network 过滤器中输入 `install/logs` 进行筛选
28. 验证存在一个 `GET /api/install/logs?installId=xxx` 类型的请求：
    - 请求方法为 `GET`
    - 请求包含 `installId` 查询参数（格式为 `install-` 开头的字符串）
    - 响应 `Content-Type` 为 `text/event-stream`
    - 响应 `Cache-Control` 包含 `no-cache`
    - 事件流类型为 `eventstream`
29. 截图（Network 中 SSE 请求详情）

### Part 4：API 响应格式验证

30. 在 Network 过滤器中输入 `update` 进行筛选
31. 验证存在一个 `POST /api/update` 请求：
    - 请求体包含 `version` 字段（值为所选版本号）
    - 响应状态码为 `200`
    - 响应体为 JSON 格式
    - 响应体中 `success` 为 `true`
    - 响应体 `data` 对象包含 `version` 字段
    - **响应体 `data` 对象包含 `installId` 字段**（格式为 `install-` 开头）
    - 响应体 `message` 包含"安装已启动"或"SSE 日志流"相关文字
32. 截图（POST /api/update 响应详情）

### Part 5：安装状态转换验证（模拟场景）

> 以下步骤基于 SSE 事件流的理论行为验证。由于不执行真实安装，
> 可通过开发者工具 Console 或 Network 中观察 EventStream 事件来确认。

33. 验证关闭按钮在安装过程中处于禁用状态（显示"安装中..."）
34. 按 Escape 键尝试关闭对话框
35. 验证对话框无法关闭（安装进行中禁止关闭）
36. 验证出现 toast 提示"安装过程中无法关闭对话框，请等待安装完成"

37. 如果安装日志对话框仍打开，等待安装超时或手动关闭
