## 前置条件
- 服务端已启动且可通过 http://localhost:9999 访问
- Dashboard 页面可正常加载

## 测试步骤

### 1. 打开端口配置对话框
1. 打开页面 http://localhost:9999/dashboard
2. 等待页面加载完成（loading 状态消失）
3. 找到并点击"配置服务端端口"按钮（位于状态卡片区域）
4. 截图（对话框打开状态）

### 2. 验证对话框内容
1. 验证对话框标题为"配置服务端端口"
2. 验证对话框描述文字为"请输入服务端端口号，系统将尝试连接。"
3. 验证对话框包含端口号输入框（placeholder 显示当前端口号，如 "9999"）
4. 验证对话框底部有"取消"和"保存"按钮
5. 截图（完整对话框界面）

### 3. 验证连接状态影响按钮文字
1. 观察保存/确认按钮的文字：
   - 如果当前显示"已连接"状态，按钮应显示"保存并重启"
   - 如果当前显示"未连接"状态，按钮应显示"保存并连接"
2. 记录当前按钮文字

### 4. 验证输入验证（可选：边界情况）
1. 清空端口号输入框
2. 输入无效值（如 "abc"、"-1"、"0"、"65536"）
3. 点击保存/确认按钮
4. 验证表单校验错误提示正常显示（如"请输入有效的端口号"或类似提示）
5. 输入有效端口号恢复测试环境

### 5. 完整端口配置流程（使用当前端口验证保存逻辑）
1. 在端口号输入框中输入当前端口号（如 "9999"，与当前值相同）
2. 点击保存/确认按钮
3. 等待操作完成
4. 验证对话框自动关闭
5. 验证 Dashboard 页面仍正常显示（未因操作异常）
6. 截图（操作完成后的 Dashboard 状态）

### 6. 清理
1. 关闭所有可能打开的对话框
2. 截图（最终状态）
