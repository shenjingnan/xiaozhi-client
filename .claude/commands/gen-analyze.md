# 问题分析

现在我有一个问题，问题描述如下：

<problem>$1</problem>

## 分析目标

请针对这个问题进行深入、全面的技术分析，重点关注问题的本质和解决方案的可行性。

## 分析流程

### 第一步：理解问题

1. **提取关键信息**

   - 识别核心问题和相关需求
   - 理解问题涉及的技术领域
   - 明确问题的影响范围

2. **收集上下文**
   - 探索相关的代码实现
   - 了解项目的技术栈和架构
   - 查找相关的文档和配置

### 第二步：深度分析

请从以下维度进行分析：

1. **现状分析**

   - 当前实现的详细说明
   - 存在的问题和挑战
   - 性能、维护性等方面的评估

2. **技术分析**

   - 架构设计合理性
   - 技术选型的优劣
   - 代码质量和工程实践

3. **解决方案评估**

   - 可能的解决方案
   - 各方案的优缺点对比
   - 实施难度和风险评估

4. **MCP 相关分析**（如适用）
   - 对 MCP 协议的影响
   - Endpoint 连接管理
   - 传输层适配器考虑

### 第三步：形成结论

1. **分析总结**

   - 问题核心原因
   - 关键发现和洞察
   - 解决建议

2. **后续建议**
   - 优先级排序
   - 实施建议
   - 风险提示

## 输出要求

1. **内容要求**

   - 使用中文编写
   - 客观、中立的分析态度
   - 基于实际代码和事实
   - 突出深度和洞察力

2. **格式要求**

   - 使用 Markdown 格式
   - 适当使用 Mermaid 图表
   - 代码示例使用项目路径别名
   - 结构清晰，逻辑连贯

3. **文件命名**
   - 存放在项目根目录
   - 格式：`分析主题-分析报告.md`
   - 示例：`endpoint-连接问题分析.md`

## 质量保证

- 分析前可运行 `pnpm check:all` 了解项目状态
- 代码示例使用 `@/xxx` 路径别名格式
- 确保所有用户界面文本使用中文
- 结论要有数据或事实支撑

## 完成后

1. 提供分析报告的文件名
2. 总结核心发现
3. 列出关键建议

---

## 分析提示

- **深度优先**：追求分析的深度而非广度
- **问题导向**：始终围绕要解决的问题
- **实用性强**：提供可操作的建议
- **技术准确**：确保技术分析的正确性
